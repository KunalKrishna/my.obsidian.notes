# OSI Model

Open Systems Interconnection model 
- **a theoretical framework** that breaks down the data encapsulation process into seven layers
- Each layer describes the information included in the message as **a header**.
- The OSI model remains widely used to teach networking and explain how protocols function. 
- It’s essential to know the first four layers as they are the ones that concern network engineers the most.

![[osi-protocol.png]]

 7 layers of the [OSI model](https://www.networkworld.com/article/964816/the-osi-model-explained-and-how-to-easily-remember-its-7-layers.html), use the mnemonic:
 **Bottom to Top (Layer 1-7):** ==**"Please Do Not Throw Sausage Pizza Away**==**"** 
 1. Physical 
 2. Data Link 
 3. Network 
 4. Transport 
 5. Session 
 6. Presentation 
 7. Application 

**Top to Bottom (Layer 7-1):** ==**A**ll **P**eople **S**eem **T**o **N**eed **D**ata **P**rocessing.==
- Application
- Presentation
- Session
- Transport
- Network
- Data Link
- Physical

**Key Details for Each Layer** 
- **Layer 7 - Application:** User interface (HTTP, FTP). 
- **Layer 6 - Presentation:** Data formatting, encryption. 
- **Layer 5 - Session:** Manages connections. 
- **Layer 4 - Transport:** End-to-end connections, reliability (TCP/UDP). 
- **Layer 3 - Network:** Logical addressing and routing (IP). 
- **Layer 2 - Data Link:** Physical addressing (MAC addresses, switches). 
- **Layer 1 - Physical:** Cables, hubs, raw data bits.
